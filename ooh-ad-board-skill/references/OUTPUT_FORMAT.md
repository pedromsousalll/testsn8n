# Output Format: How to package the final deliverable

## Goal
Define a consistent, export-friendly structure so your output can be pasted into:
- Google Slides / PowerPoint
- Notion / Confluence
- PDF export workflows

## 1) Standard output format (Markdown)
Use this exact structure:

- Title block (1–3 lines)
- Slides:
  - `Slide 1 — <Title>`
    - Bullet 1
    - Bullet 2
    - Bullet 3
    - `Speaker notes:` <notes>

Repeat for all slides.

## 2) Required final sections
Always append:

### Assets checklist
A bulleted list of required materials, for example:
- Logos (SVG/PNG), brand colors, fonts
- Product screenshots / photos
- Offer terms / legal line
- QR destination URL + tracking parameters

### Next steps
A short, numbered list of actions (3–7 items), for example:
1. Confirm objective + audience
2. Finalize slide count + tone
3. Draft copy → design → review
4. Approvals (brand/legal/media owner)
5. Export + QA

## 3) Optional packaging modes
If the user asks for a specific “final document” style, provide one of these modes.

### A) Slide-ready copy mode
- Keep bullets short (max ~12 words)
- Add an "On-slide copy" line if needed
- Put longer rationale only in speaker notes

### B) One-pager brief mode
Provide headings:
- Objective
- Audience
- Key message
- Plan
- KPIs
- Timeline
- Owners

### C) Presentation spec mode
Include:
- Slide count
- Aspect ratio (16:9 default)
- Design style (minimal, bold, etc.)
- Font pairing (placeholder if unknown)

## 4) Filename + versioning convention
Use a deterministic name so it’s easy to export and track:
- `<project>_<type>_<YYYY-MM-DD>_v<major>.<minor>`
Examples:
- `flash_sale_ooh_2026-01-29_v1.0`
- `q1_growth_marketing_2026-01-29_v1.0`

## 5) Output constraints by presentation type
### Ad board / OOH
- Prefer 6–8 slides
- Enforce the “3-second rule” in copy
- Include specs/placements + measurement

### Marketing
- Prefer 8–12 slides
- Include objectives, strategy, channels, KPIs, timeline
